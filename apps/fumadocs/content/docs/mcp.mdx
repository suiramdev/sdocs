Use MCP when your IDE or agent supports MCP servers.

<Callout title="In development" type="warning">
  This feature is still in development and may not work yet, or may behave
  differently than intended.
</Callout>

## 1) Configure your preferred tool

<Tabs items={["Cursor", "Other MCP tools"]}>
  <Tab>
    Install with one click:

    <McpCursorInstallButton />

    Or paste this server config in Cursor MCP settings:

    ```jsonc
    {
      "mcpServers": {
        "sdocs": {
          "url": "https://sdocs.suiram.dev/api/v1/mcp"
        }
      }
    }
    ```

  </Tab>
  <Tab>
    In your tool's MCP settings (Windsurf, Cline, Zed, Claude Desktop, and similar), add:

    ```jsonc
    {
      "mcpServers": {
        "sdocs": {
          "url": "https://sdocs.suiram.dev/api/v1/mcp"
        }
      }
    }
    ```

  </Tab>
</Tabs>

## 2) Restart your tool

After saving the config, restart your IDE/agent so it can discover the `sdocs` MCP server.

## 3) Verify with a quick prompt

```text
How do I implement Player movement in S&Box C#, and what is the exact API signature?
```

## What this gives your agent

Once connected, your tool can:

- `search_docs` - search the s&box documentation by keyword
- `describe_entity` - get a high-level description of an API entity
- `get_signature` - fetch method and property signatures for an API entity
- `answer_question` - answer natural-language questions grounded in the docs

Your IDE usually exposes these as built-in tools/actions after MCP connection.
