## Quick start for IDEs (recommended)

Add or merge the following into your MCP configuration file (for example in Cursor, Windsurf, Zed, or any other MCP-aware IDE or agent).

If you are using Cursor, you can click the button to install the server automatically, or paste the config manually from the block below:

<McpCursorInstallButton />

```jsonc
{
  "mcpServers": {
    "sdocs": {
      "type": "sse",
      "url": "https://sdocs.suiram.dev/api/v1/mcp",
    },
  },
}
```

After saving the config, restart your IDE or agent so it can discover the new `sdocs` MCP server and its tools.

## What tools your agent gets

Once connected, your MCP client can call these tools:

- `search_docs` – search the s&box documentation by keyword
- `describe_entity` – get a high-level description of an API entity
- `get_signature` – fetch method and property signatures for an API entity
- `answer_question` – answer natural-language questions grounded in the docs

Your IDE or agent will usually surface these as tools or actions; no extra wiring is needed beyond the MCP config above.

## Direct JSON-RPC usage (optional)

If you are building your own MCP client, you can talk to the same endpoint directly over JSON-RPC.

### 1) Initialize

```bash
curl -X POST "https://sdocs.suiram.dev/api/v1/mcp" \
  -H "content-type: application/json" \
  -H "accept: application/json, text/event-stream" \
  -d '{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "initialize",
    "params": {
      "protocolVersion": "2025-03-26",
      "capabilities": {},
      "clientInfo": {
        "name": "my-client",
        "version": "1.0.0"
      }
    }
  }'
```

Then send:

```bash
curl -X POST "https://sdocs.suiram.dev/api/v1/mcp" \
  -H "content-type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "notifications/initialized",
    "params": {}
  }'
```

### 2) List tools

```bash
curl -X POST "https://sdocs.suiram.dev/api/v1/mcp" \
  -H "content-type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 2,
    "method": "tools/list",
    "params": {}
  }'
```

### 3) Call a tool

Example `search_docs` call:

```bash
curl -X POST "https://sdocs.suiram.dev/api/v1/mcp" \
  -H "content-type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 3,
    "method": "tools/call",
    "params": {
      "name": "search_docs",
      "arguments": {
        "query": "player movement",
        "limit": 5
      }
    }
  }'
```

The tool result includes:

- `content` – human-readable text blocks
- `structuredContent` – machine-friendly data you can pass back to the model

### 4) Optional variables for better RAG answer quality

```bash
OPENAI_API_KEY="..."
API_RAG_MODEL="gpt-4.1-mini"
```

If no `OPENAI_API_KEY` is set, answers still work using grounded fallback output.
